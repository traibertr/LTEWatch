Register a new gpio in the controller class.

This method registerGpio MUST be used to register a new Gpio. In fact, this method just save the pointer to the instance of a gpio in an array to keep a reference and be able to detect a single action on any gpio's instance.

Parameters
gpioPointer to the new gpio to register.
useTimerSpecify if the GpioController will add a delay before notifying any gpio's state change.
debouncDelayThe debouncer's delay in [ms].
eventHandlerPointer to the method handling the GPIO Task and Events
Returns
true if the registration succeeds or false otherwise.

◆ stateChanged()

void GpioController::stateChanged ( uint8_t  index,
Gpio::eGpioState  gpioState 
)

Notify the controller that a gpio has been pressed.

Every time a gpio is released after the Debounce delay, the controller is notified. This method needs the index of the gpio and which state is applied.

gpioState can take the values :

  • GpioShortClick: Gpio pressed for a short delay.
  • GpioLongClick: Gpio pressed for a long delay.
  • GpioVeryLongClick: Gpio pressed for a very long delay. Other value should not be considered here.
Parameters
indexThe identifier of the gpio.
gpioStateThe kind of pressed applied on the gpio.

◆ timerEventHandler()

void GpioController::timerEventHandler ( )

Timeout's handler for the SoftDevice timer (Gpio's Handler timer).

This function will be called each time the Button's handler interval timer expires. This function will wake up the CPU and increment a variable. It is also the callback method of the app_timer used by the button controller.
