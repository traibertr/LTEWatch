Detailed Description

XFThread represents the instance executing the behaviour.

XFThread representing the instance executing the behaviour. In a OS less XF, only one instance of XFThread gets created (see XF::_mainThread). In a XF interfacing an OS, multiple instances can be made, each interfacing the real OS thread.

Adds a new time-out to be handled. The XFThread will forward the time-out information to the time-out manager which is responsible to handle all time-outs.

Parameters
timeoutIdThe time-out id known by the reactive parameter. Is needed by the reactive part to uniquely identify the time-out.
intervalThe time in milliseconds or timer's ticks unit to wait until the time-out expires.
pReactiveThe reactive instance where to inject the time-out when it time-outs.
